Herança

Herança permite que uma classe erde atributos e métodos de outra classe, promovendo reutilização de código.

class Animal{
  void emitirSom(){
    System.out.println("Som de animal");
  }
}
class Cachorro extends Animal{
  void emitirSom(){
    System.out.println("Au au");
  }
}
class Animal:
  def emitir_som(self):
    print("Som de animal")

class Cachorro(Animal):
  def emitir_som(self):
    print("Au au")
class Animal {
public:
  virtual void emitirSom() {
    cout << "Som de animal";
  }
};
class Cachorro : public Animal {
public:
  void emitirSom() override {
    cout << "Au au";
  }
};

A classe Cachorro herda da classe Animal e sobrescreve o método emitirSom()

Por que usar Herança?

A herança é um dos pilares da POO e permite que você reaproveite código já escrito, evitando duplicação e facilitando a manutenção do sistema.

  • Evita repetição de código
  • Facilita a extensão de funcionalidades
  • Organiza hierarquias de classes
  • Facilita o polimorfismo

Interface

Uma interface define um contrato que as classes devem cumprir, estabelecendo quais métodos precisam ser implementados.

O que é uma Interface?

Em Java, uma interface é um tipo abstrato que define um conjunto de métodos que uma classe deve implementar, sem fornecer a implementação.

interface Animal {
  void emitirSom();
}
class Cachorro implements Animal {
  public void emitirSom() {
    System.out.println("Au au");
  }
}
